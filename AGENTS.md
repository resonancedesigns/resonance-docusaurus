# Repository Guidelines

## Project Structure & Module Organization
- `docs/`: Markdown/MDX docs; sidebar autogenerated via `sidebars.ts`.
- `src/`: React/TypeScript pages, theme overrides, hooks, utilities, tests (next to code).
- `static/`: Static assets served at site root (e.g., `themes/*.css`).
- `config/`: YAML sources consumed by pre-build.
- `scripts/`: TypeScript utilities (pre-build, schema validation).
- `data/`: Generated JSON + `index.ts` from YAML (do not edit).
- `artifacts/`: Production build output.
- Key config: `docusaurus.config.ts`, `sidebars.ts`, `tsconfig.json`, `eslint.config.js`.

## Build, Test, and Development Commands
- `pnpm dev`: Watch `config/**/*.yml`, run pre-build, start Docusaurus.
- `pnpm start`: Local dev server (runs pre-build once).
- `pnpm build`: Pre-build then Docusaurus static build to `./artifacts`.
- `pnpm serve`: Preview a production build.
- `pnpm test` / `pnpm test:run` / `pnpm test:ui`: Vitest (CLI/UI); `pnpm test:coverage` for reports.
- `pnpm lint` / `pnpm lint:fix`, `pnpm format` / `pnpm format:check`, `pnpm quality` for full checks.

## Coding Style & Naming Conventions
- TypeScript + React function components and hooks.
- Prettier: single quotes, semicolons, no trailing commas; 2-space indent.
- ESLint TS rules (relaxed strictness); fix warnings before commit.
- Naming: `PascalCase` components, `camelCase` vars/functions, `kebab-case` files.

## Testing Guidelines
- Framework: Vitest + jsdom; setup in `vitest.setup.ts` with Docusaurus mocks.
- Tests colocated: `*.test.ts` / `*.test.tsx` alongside source.
- Coverage thresholds: lines 75, functions 70, branches 60, statements 75.
- Run locally: `pnpm test` or `pnpm test:run` before PRs.

## Commit & Pull Request Guidelines
- Conventional Commits (e.g., `feat: add search bar`).
- Husky: `pre-commit` (lint), `commit-msg` (length + guidance), `post-commit` (push).
- PRs: include summary, linked issues, screenshots for UI changes, and validation steps.

## Security & Configuration Tips
- Engines: Node >= 18, pnpm >= 8.
- Edit `config/**/*.yml` under `pnpm dev` (auto pre-build) or run `pnpm prebuild`.
- Do not edit generated files in `data/`; source truth is `config/`.
